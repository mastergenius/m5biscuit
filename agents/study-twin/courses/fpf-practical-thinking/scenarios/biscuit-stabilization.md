# Biscuit Stabilization Scenario

The course should use live project material because that is where the method becomes useful.

## Scenario Threads

`File Transfer security`
: The device exposes file access only in explicit transfer mode and behind a token. Future work is
  hardening for untrusted networks.

`StudyPacks`
: The product signal moved from generic flash cards to personalized adult learning units with
  offline-first runtime and Mac-side twin generation.

`Power and wake`
: Wake behavior and deep sleep must be treated as device-specific operational constraints.

`WDT log noise`
: The WDT spam should be recorded as a risk because it can hide real watchdog failures, but it is not
  the current focus.

## What This Tests

- Can the learner name the real signal?
- Can they keep scope narrow without forgetting adjacent risks?
- Can they write an update that reduces uncertainty?
- Can they avoid pairwise device integration as a distraction from the stronger current product
  signal?
