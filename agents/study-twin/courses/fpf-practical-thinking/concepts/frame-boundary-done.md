# Signal, Boundary, Done

FPF is useful when it prevents work from turning into fog. The minimal practical move is:

```text
signal -> boundary -> observable done
```

`signal`
: What observation, opportunity, question, or pain triggered the work?

`boundary`
: What is in scope, what is out of scope, and what context are we solving for?

`done`
: What observable state of the world tells us this pass is complete?

## Example

Task: improve StudyPacks.

- Signal: toy flash cards do not help the real goal of practical adult learning.
- Boundary: design useful French and FPF packs plus offline sync semantics.
- Out of scope: full native Mac/iPad product today.
- Done: packs validate, upload, open on device, and produce review logs.

## Teaching Rule

Prefer plain operational language over method names. If the learner can use the move tomorrow, the
terminology can come later.
