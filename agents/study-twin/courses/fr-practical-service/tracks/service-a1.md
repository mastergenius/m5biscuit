# Service A1 Track

The track uses one real service situation as a compression device. A haircut is useful because it is
short, constrained, and unforgiving enough to expose missing language control.

## Teaching Sequence

1. Treat the situation as a service scenario, not as a long dialogue.
2. Build the appointment request from slots.
3. Add time as a separate slot.
4. Add preference and degree words.
5. Add repair phrases for misunderstanding.
6. Turn the pieces into a personal script.
7. Transfer the same pattern to other services.

## Competency Targets

`fr.competency.book-service-appointment`
: The learner can ask for an appointment for a named service and suggest a day or time window.

`fr.competency.describe-basic-preference`
: The learner can state a simple desired result and a boundary such as "not too short".

`fr.competency.repair-understanding`
: The learner can ask for repetition, slower speech, or demonstration instead of silently agreeing.

`fr.competency.compose-slot-sentence`
: The learner can combine operator, action/object, time/place, and constraint slots.

## Goldilocks Boundary

The first boundary is not "knows many words". It is "can assemble and adapt a short working phrase".
The stretch item should change the time, service, or constraint while keeping the same pattern.
